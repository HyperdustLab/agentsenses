# Security Policy

## Supported Versions

This project is currently in an early release line (`0.x`).

Security fixes are provided on the latest commit on `main`. If stable release branches are introduced later, this policy will be updated with a version support table.

## Reporting a Vulnerability

Please do not open a public issue for suspected security vulnerabilities.

Instead, report privately by emailing the maintainers (or the project owner) with:

- A clear description of the issue and affected component
- Reproduction steps or proof-of-concept (if available)
- Potential impact and any known mitigations
- Your preferred disclosure contact and attribution name (optional)

We aim to acknowledge reports within 3 business days and provide a remediation plan or status update as soon as triage is complete.

## Disclosure Process

- We will validate and triage the report.
- We will prepare and test a fix.
- We will coordinate disclosure timing with the reporter when possible.
- We will publish release notes in `CHANGELOG.md` for patched versions.
